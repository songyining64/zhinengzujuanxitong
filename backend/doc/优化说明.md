# 近期优化说明

## 后端

1. **答卷摘要接口** `GET /api/exam/record/{recordId}/summary`  
   一次返回分数、及格、名次（教师/已发布成绩时对学生可见）、切屏次数等，减少前端多次请求。

2. **Actuator 健康检查**  
   依赖 `spring-boot-starter-actuator`，暴露 `GET /actuator/health`（无需登录），便于部署与监控探活。

## 前端

1. **`src/api/http.ts`**  
   统一 Axios：自动带 `Authorization`、解析业务 `code`、401 清理 token 并跳转登录。

2. **登录**  
   成功提示、保存 `role` 到 `localStorage`；错误由拦截器提示。

3. **侧栏菜单**  
   从 `GET /api/system/menu/tree` 动态渲染，与后端角色权限一致；未配置路由的菜单进入占位页 `Placeholder.vue`。

4. **`useExamAntiCheat`**  
   组合式函数：监听 `visibilitychange` 防抖后调用 `/api/exam/record/{id}/blur`，供答题页接入。

5. **答题页**（`/exam/take` 列表 → `/exam/take/:recordId` 作答）  
   支持单选/多选/判断/填空/简答、顶部倒计时与切屏提示、草稿自动保存（约 8 秒防抖 + 每 30 秒定时）、交卷与结果展示。非学生账号（除 ADMIN）无法进入答题路由。

6. **答卷摘要 DTO**  
   `summary` 接口增加 `startedAt`、`deadlineAt`，便于答题页倒计时。

## P1 已实现（需执行数据库迁移）

在**已有数据库**上依次执行：

1. `backend/doc/migration_v2_p1.sql`（`question.review_status`、组卷模板与审计等）  
2. `backend/doc/migration_v3_question_version_dedup.sql`（`question.version_no`、`question_version` 历史快照表）

全新建库可直接用更新后的 `schema.sql`。

### 试题

- `GET /api/question?reviewStatus=` 分页筛选审核状态  
- `POST /api/question/batch` 批量改知识点/难度  
- `POST /api/question/import?courseId=` Excel 导入  
- `GET /api/question/export?courseId=` 导出 Excel  
- `GET /api/question/import/template` 下载导入模板  
- `POST /api/question/{id}/submit-review` 提交审核（教师）  
- `POST /api/question/{id}/approve`、`/reject` 审核通过/驳回（仅 ADMIN）

### 试题版本与查重

- `POST /api/question/dedup-check`：题干 + 选项 JSON 综合相似度（默认阈值 0.78，最多比对 5000 题）  
- `GET /api/question/{id}/versions`：历史快照列表；`GET /api/question/{id}/versions/{versionNo}`：某一版快照  
- 修改试题、批量改、审核状态变更时会**自动归档**上一版到 `question_version`

### 组卷

- `PaperAutoGenRequest`：`randomPool`（全课程随机池）、`fixedQuestionIds`（必考题前置）；自动组卷仅选用 **已发布**（`review_status=PUBLISHED`）试题。  
- `POST /api/paper/from-template/{templateId}` 按模板生成试卷  
- `GET /api/paper/generation-logs?courseId=` 组卷审计记录  
- `GET|POST|PUT|DELETE /api/paper-template` 组卷模板 CRUD
