# 权限、成绩发布与防作弊说明

## 1. 菜单 `perms` 与接口对齐

- 启动时若 `sys_menu` 为空，由 `MenuAndPermissionSeed` 写入菜单及 `sys_role_menu`，`perms` 与 Controller 上 `@PreAuthorize(... hasAuthority('xxx'))` 一致。
- 登录后 JWT 解析出的用户会携带 **ROLE_*** 与 **菜单上的权限字符串**（见 `UserDetailsServiceImpl` + `SysMenuServiceImpl#listGrantedPermsForRole`）。
- **课程维度的数据隔离**仍由 `CoursePermissionService`（教师/选课学生）在业务层校验；**菜单权限**控制「能否调用某类接口」，二者组合满足课题常见的「角色 + 课程数据权限」要求。

权限标识示例：`course:read`、`course:manage`、`exam:manage`、`exam:student`、`exam:analytics`、`wrongbook:read` 等。

## 2. 及格线与成绩发布

- 考试表字段：`pass_score`（及格线，可为空）、`score_published`（是否已发布成绩）。
- 创建/更新考试时可在请求体中携带 `passScore`、`scorePublished` 等（见 `ExamCreateRequest`）。
- 教师发布成绩：`POST /api/exam/{id}/publish-score`；撤回：`POST /api/exam/{id}/unpublish-score`。
- **学生**在 `score_published = 0` 时调用成绩统计/排名相关接口会返回 **403**（`ExamAnalyticsServiceImpl#assertStudentScorePublishedIfNeeded`），避免半成品分数引发纠纷。
- 排名 DTO 增加 `passed`（相对及格线），概览增加 `passCount` / `passScore` / `scorePublished`。

**已有库请执行** `doc/migration-20250317.sql` 以添加列。

## 3. 组卷难度比例

- 自动组卷请求 `PaperAutoGenRequest` 增加可选字段 `difficultyWeights`（如 `EASY:3, MEDIUM:5, HARD:2`），与原有 `countByType` 配合：在**每种题型内**按权重分配易/中/难题量。
- 题库 `question.difficulty`：1=易，2=中，3=难；未填难度的题目在按难度抽题时归入「易」档（与难度 1 一起抽取）。

## 4. 考试防作弊（乱序 + 切屏）

- **题目乱序**：`exam.shuffle_questions`（默认开启）。开考时为该生生成 `shuffle_seed` 与 `question_order_json`。
- **选项乱序**：`exam.shuffle_options`（默认开启）。客观题展示乱序选项；服务端将学生提交的「展示选项字母」转换为原始标号再判分与存储。
- **切屏**：`exam.switch_blur_limit`（null 或 0 表示不限制）。学生失焦/切屏时调用 `POST /api/exam/record/{recordId}/blur`，服务端累加 `switch_blur_count`，达到上限则**自动交卷**（使用已保存答案，不再做展示→原始二次转换）。

前端建议：在答题页监听 `document.visibilitychange`、`window.blur`，防抖后调用 `/blur` 接口。

## 5. 论文「展望」表述建议

若需强调可扩展性，可写：在「难度比例组卷」基础上，可进一步做「题型 × 难度」联合矩阵、知识点覆盖约束与多目标优化组卷等。
