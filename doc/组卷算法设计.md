# 组卷算法设计

## 1. 目标

在指定课程、知识点范围内，按「题型 → 数量」或「题型 + 难度权重」自动选题，生成试卷（`paper` + `paper_question`），并可选题目去重、随机种子可复现。

## 2. 输入（`PaperAutoGenRequest`）

| 字段 | 含义 |
|------|------|
| `courseId` | 课程 ID |
| `knowledgePointIds` | 知识点根节点 |
| `includeKnowledgeDescendants` | 是否包含子知识点 |
| `countByType` | 题型 → 数量，如 `SINGLE:5`、`MULTIPLE:3` |
| `scorePerQuestion` | 每题分值（默认 10） |
| `randomSeed` | 随机种子（可选，缺省用系统纳秒） |
| `dedup` | 是否跨题型去重（默认 true） |
| `difficultyWeights` | 可选，如 `EASY:3, MEDIUM:5, HARD:2` |

**难度与题库**：`question.difficulty`：`1`=易，`2`=中，`3`=难；未填 `difficulty` 的题在「易」档与 `1` 一起抽取。

## 3. 抽题流程

1. **知识点集合**：展开为 `kpFilter`（含子孙节点）。
2. **按题型遍历** `countByType`：
   - **未配置难度权重**：按题型从题库池随机打乱后取 `need` 道，去重。
   - **配置难度权重**：  
     - 调用 `PaperDifficultyAllocator.allocateNeedByWeights(need, wE, wM, wH)` 将 `need` 拆成 `[易, 中, 难]` 三道题量，三者之和为 `need`。  
     - 每档在「题型 + 难度 + 知识点」条件内打乱后取题；易档包含 `difficulty IS NULL OR difficulty=1`。
3. **不足**：任一档题量不足则抛业务异常，提示题型与难度。
4. **落库**：写入 `paper`、`paper_question`，`rules_json` 记录完整规则快照。

## 4. 无重复试题

- `dedup=true` 时，`picked` 集合记录已选 `question_id`，同一题不会出现在多题型中。

## 5. 可扩展（论文展望）

- 题型 × 难度联合矩阵（显式指定每种组合数量）。
- 知识点覆盖约束（每知识点至少 N 题）。
- 多目标优化（难度、区分度、曝光均衡）。
