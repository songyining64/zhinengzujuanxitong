# 核心业务深化设计（贴合组卷场景）与落地路线图

> 本文档与当前仓库 `backend/doc/schema.sql`、组卷实现（`PaperServiceImpl`、`PaperDifficultyAllocator`）对齐，用于**课题扩展、答辩与迭代规划**。完整开发量较大，建议按阶段实施。

---

## 一、题库管理能力升级

### 1.1 批量操作（导入 / 批量编辑 / 导出）

| 能力 | 说明 | 技术要点（与现有栈契合） |
|------|------|--------------------------|
| **批量导入** | Excel 模板（推荐首版）：列含题型、题干、选项 JSON、答案、知识点编码、难度等 | 后端 **EasyExcel**（项目已依赖）流式读取；模板下载 `GET /api/question/import/template`；导入 `POST /api/question/import` 异步任务 + 错误行报告 |
| **Word 导入** | 结构化差，可作为二期：固定样式解析或用第三方转 HTML 再入库 | Apache POI / docx4j；优先保证 Excel 闭环 |
| **批量编辑** | 按筛选条件批量改 `knowledge_point_id`、`difficulty` | `PATCH /api/question/batch` + 条件 DTO（courseId + filters + patch） |
| **批量导出** | 与导入列一致，便于往返 | EasyExcel 写出；大数据量用分页导出或异步文件 + 下载链接 |

**现状**：单条 CRUD 已有；需在 `question` 表或扩展表上支持**导入批次号**便于回滚。

---

### 1.2 试题多维度管控

#### A. 审核流程（教师提报 → 管理员审核入库）

建议新增状态机，与现有 `question.status` 扩展或拆分：

| 状态 | 含义 |
|------|------|
| `DRAFT` | 教师草稿 |
| `PENDING_REVIEW` | 已提报待审 |
| `PUBLISHED` | 审核通过入库（仅此类参与组卷） |
| `REJECTED` | 驳回 |

**表**：`question_review_log`（`question_id`, `from_status`, `to_status`, `operator_id`, `comment`, `time`）。

**接口**：教师 `submitReview`；管理员 `approve` / `reject`。组卷 SQL 增加 `status = PUBLISHED`（或等价枚举）。

#### B. 试题版本管理

**思路**：题干/答案变更不覆盖原行，插入 `question_version`（`question_id`, `version`, `stem_snapshot`, `options_snapshot`, `answer_snapshot`, `editor_id`, `created_at`）；`question` 表保留 `current_version_id` 或 `version_no`。

**组卷**：`paper_question` 存 `question_id` + `question_version_id`（快照），避免历史试卷随改题变化。

#### C. 试题查重（相似度）

| 层级 | 策略 |
|------|------|
| 快速 | 题干 **SimHash** / **MinHash** + 海明距离阈值；同课程内检索 |
| 精细 | 选项集合 Jaccard；题干分词后向量（可选小型中文 embedding 服务） |
| 知识点 | 同知识点下重复率加权 |

**接口**：`POST /api/question/dedup-check` 返回相似题列表；导入时批量调用。

---

### 1.3 多媒体试题（图片 / 公式 / 音频）

**存储**：题干、解析、选项不限于纯文本，建议使用 **JSON 富文本结构** 或统一约定：

```json
{
  "blocks": [
    { "type": "text", "content": "题干文字" },
    { "type": "image", "fileId": 123, "alt": "" },
    { "type": "latex", "content": "E=mc^2" },
    { "type": "audio", "fileId": 456 }
  ]
}
```

- **图片/音频**：沿用 `file_resource` + URL，前端展示时拼接；大文件可走对象存储（OSS/MinIO）。
- **公式**：后端存 LaTeX 字符串；前端 **KaTeX** / **MathJax** 渲染（答题页、预览页）。
- **题型扩展**：在 `QuestionTypeEnum` 中增加 `ESSAY`（论述）、`CALC`（计算）等，判分规则与 `QuestionAnswerMatcher` 同步扩展。

**现状**：`stem`/`options_json` 为 TEXT；可渐进式兼容「纯文本」与「blocks JSON」两种形态（加字段 `content_format`：`PLAIN` | `RICH_JSON`）。

## 二、组卷算法与策略优化

### 2.1 多模式组卷

| 模式 | 含义 | 实现要点 |
|------|------|----------|
| **基础模式**（已有） | 知识点 + 题型 + 可选难度权重 | `generateAuto` + `rules_json` |
| **固定试题组卷** | 必考题 ID 列表 + 其余自动补齐 | `PaperAutoGenRequest` 增加 `fixedQuestionIds`；先锁定再池内抽题 |
| **随机组卷** | 弱化知识点约束，扩大池随机 | `randomMode=true` + 可选仅课程级过滤 |
| **个性化组卷** | 依据错题本 / 薄弱知识点加权 | 读 `wrong_book` + 知识点正确率；**加权随机**或 **多臂老虎机** 简化版 |

**接口**：`mode` 枚举：`AUTO_BASIC / AUTO_FIXED / AUTO_RANDOM / AUTO_PERSONALIZED` 写入 `paper.rules_json`。

---

### 2.2 性能与模板、缓存

| 项 | 说明 |
|----|------|
| **减少全表扫描** | 题目池按 `(course_id, type, knowledge_point_id, difficulty, status)` **复合索引**；抽题仅用 `WHERE` + `IN` + `LIMIT` 预筛选再 `ORDER BY RAND()` 或 **内存随机**（先查 ID 列表再随机） |
| **组卷模板** | 表 `paper_template`（`name`, `course_id`, `rules_json`, `creator_id`）；一键生成 `POST /api/paper/from-template/{id}` |
| **Redis 缓存** | Key：`paper:gen:{hash(rules)}` → 生成结果 `paper_id` 或题目 ID 列表 TTL；**仅当** `random_seed` 相同且规则不变时命中（项目已具备 Redis 可选配置） |

---

### 2.3 组卷约束强化

| 约束 | 实现 |
|------|------|
| **试题来源范围** | `rules_json` 增加 `chapterIds` / `yearRange` / `tags`；`question` 表扩展 `source_year`、`tags` 等 |
| **同卷无重复** | 已有 `dedup`；扩展为「跨课程共享题库」时的全局去重 |
| **多次组卷低重复** | 记录学生已抽题集合 `student_paper_question` 或会话内 `seen_question_ids`，加权降低已见题概率 |
| **总分 / 时长** | 总分由 `paper_question.score` 汇总；考试时长由 `exam.duration_minutes` 已有；可增加「根据总分量自动建议时长」的启发式 |

## 三、试卷全生命周期管理

### 3.1 预览与编辑

- **在线预览**：前端按 `paper_question` 顺序渲染，与答题页共用排版组件（只读）。
- **手动调整**：`PATCH /api/paper/{id}/questions` 增删改顺序与分值（教师权限）。
- **排版**（字体/间距/页码）：导出阶段用模板控制；前端预览 CSS 变量一套即可。

### 3.2 导出多样化

| 格式 | 技术 |
|------|------|
| **Word** | Apache POI / docx4j，模板占位符替换 |
| **PDF** | OpenPDF / Flying Saucer + HTML 模板，或 wkhtmltopdf 侧车 |
| **Excel 阅卷版** | EasyExcel：列=学号、题号、得分 |
| **答题卡** | 填涂卡 PDF 或独立模板生成 |

### 3.3 组卷记录追溯（审计）

**表**：`paper_generation_log`（`id`, `paper_id`, `operator_id`, `mode`, `rules_json`, `duration_ms`, `client_ip`, `created_at`）。

**接口**：教师/管理员 `GET /api/paper/generation-logs` 条件查询 + 导出 Excel。

## 四、与当前系统的映射（差距清单）

| 模块 | 当前能力 | 目标能力 |
|------|----------|----------|
| 题库 | 单条维护、导入未做 | 批量导入导出、审核、版本、查重 |
| 题干 | TEXT | 富文本/多媒体 JSON |
| 组卷 | 知识点+题型+难度权重 | 固定题/随机/个性化、模板、缓存、更多约束 |
| 试卷 | CRUD、详情 | 预览编辑、多格式导出 |
| 审计 | 无 | 组卷与导入操作日志 |
